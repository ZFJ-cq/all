# 个人轨迹记录仪 — 产品需求说明书（PRD）

> 版本：v1.0  
> 日期：2026-06-05  
> 产品类型：纯前端 PWA 应用（Web + 安卓 APK）

---

## 1. 项目简介

### 1.1 项目背景

个人出行场景中，用户常需记录跑步、骑行、徒步等运动轨迹，或旅行、通勤等出行路线。现有轨迹记录工具多依赖云端账号、存在隐私顾虑，且功能臃肿、需付费解锁。本项目旨在提供一款零服务器依赖、数据完全本地化的轻量轨迹记录工具，兼顾 Web 访问与安卓原生体验。

### 1.2 产品定位

- **一句话定位**：纯本地、零注册、零云端的个人轨迹记录 PWA 工具。
- **核心价值**：隐私安全（数据不上传）、零成本（地图 API 免费）、即开即用（PWA 秒装）。

### 1.3 建设目标

| 目标维度 | 具体指标 |
|---------|---------|
| 功能完整性 | 支持轨迹录制、实时绘制、历史回放、离线查看 |
| 隐私安全 | 所有数据仅存于本机 LocalStorage，无任何网络上传 |
| 部署灵活 | 支持 Web 静态托管、PWA 桌面添加、安卓 APK 打包三种使用方式 |
| 零成本运行 | 高德地图 JS API 个人免费版，无需付费、无需企业认证 |

### 1.4 适用终端

| 终端类型 | 支持方式 | 最低要求 |
|---------|---------|---------|
| 移动端浏览器 | 直接访问网页 | Chrome 80+ / Safari 14+ / Firefox 80+ |
| 安卓 APP | APK 打包安装 | Android 7.0+（需 WebView 支持） |
| 桌面浏览器 | 调试/预览用 | Chrome 80+ |

## 2. 用户说明

- **用户类型**：个人自用用户
- **用户规模**：单用户（仅记录本人出行轨迹）
- **使用场景**：
  - 跑步/骑行/徒步等运动轨迹记录
  - 旅行路线记录与回放
  - 日常通勤路线追踪
- **核心诉求**：操作简单、数据私密、无需注册登录、离线可用

## 3. 整体产品架构说明

```
┌─────────────────────────────────────────────┐
│                 用户界面层                     │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│  │ 首页地图  │ │ 轨迹录制  │ │ 历史管理  │    │
│  └──────────┘ └──────────┘ └──────────┘    │
├─────────────────────────────────────────────┤
│               PWA 能力层                      │
│  ┌──────────────┐ ┌───────────────────┐    │
│  │ manifest.json │ │ Service Worker    │    │
│  │ (安装/全屏)   │ │ (离线缓存/资源预存) │    │
│  └──────────────┘ └───────────────────┘    │
├─────────────────────────────────────────────┤
│               数据持久化层                     │
│  ┌────────────────────────────────────┐     │
│  │ LocalStorage（轨迹数据/设置项）      │     │
│  └────────────────────────────────────┘     │
├─────────────────────────────────────────────┤
│               第三方服务层                     │
│  ┌────────────────────────────────────┐     │

│  └────────────────────────────────────┘     │
└─────────────────────────────────────────────┘
```

**架构要点**：
- 纯前端静态项目，无后端服务
- HTML5 + JavaScript + CSS 单页应用
- 高德地图 JS API 提供地图渲染与定位能力
- Service Worker 实现离线缓存与资源预存
- LocalStorage 实现数据持久化
- 支持 TWA（Trusted Web Activity）方式打包安卓 APK

## 4. 功能模块需求

### 4.1 首页地图模块

#### 4.1.1 地图初始化

| 项目 | 规则 |
|------|------|
| 地图容器 | 全屏显示，占满视口 |
| 默认中心点 | 用户当前定位坐标；定位失败时默认北京（116.397428, 39.90923） |
| 默认缩放级别 | 15 |
| 地图类型 | 标准地图（AMap.Map） |
| 缩放范围 | 3-18 |
| 交互 | 支持拖拽、缩放、双击放大 |

#### 4.1.2 定位权限申请

| 项目 | 规则 |
|------|------|
| 定位方式 | HTML5 Geolocation API + 高德定位插件（AMap.Geolocation） |
| 权限申请时机 | 首次打开页面时自动申请 |
| 权限被拒处理 | 弹出提示"需要定位权限才能记录轨迹"，提供手动选择城市入口 |
| 定位精度 | 优先使用 GPS 高精度模式（enableHighAccuracy: true） |
| 超时设置 | 定位超时 10 秒 |
| 定位回调 | 定位成功后自动将地图中心移至当前位置 |

#### 4.1.3 点位展示

| 项目 | 规则 |
|------|------|
| 当前位置标记 | 蓝色圆点 + 蓝色精度圈 |
| 轨迹点位 | 沿轨迹线显示，不单独标记 |
| 标记跟随 | 录制中当前位置标记实时更新 |

---

### 4.2 轨迹录制模块

#### 4.2.1 开始记录

| 项目 | 规则 |
|------|------|
| 入口 | 首页底部悬浮"开始记录"按钮 |
| 前置条件 | 定位权限已获取 |
| 操作 | 点击按钮 → 按钮变为"暂停" → 开始采集 GPS 点位 |
| 采集频率 | 每 3 秒采集一次 GPS 坐标 |
| 采集数据 | 经度（lng）、纬度（lat）、时间戳（timestamp）、精度（accuracy） |
| 最小精度过滤 | 精度 > 100 米的点位丢弃，不存入轨迹 |
| 距离过滤 | 连续两点距离 < 2 米的点位丢弃，减少冗余 |

#### 4.2.2 暂停/停止记录

| 操作 | 行为 |
|------|------|
| 暂停 | 点击"暂停"按钮 → 停止采集点位 → 按钮变为"继续" → 保留当前轨迹数据 |
| 继续 | 点击"继续"按钮 → 恢复采集 → 按钮变为"暂停" |
| 停止 | 点击"停止"按钮 → 弹出确认对话框 → 确认后保存轨迹 → 返回首页 |
| 自动停止 | 页面切至后台超过 5 分钟 → 自动暂停采集（visibilitychange 事件监听） |

#### 4.2.3 实时轨迹线绘制

| 项目 | 规则 |
|------|------|
| 绘制方式 | AMap.Polyline 实时追加坐标点 |
| 线条颜色 | #4A90D9（蓝色） |
| 线条宽度 | 6 像素 |
| 线条样式 | 实线，圆角端点（lineJoin: round, lineCap: round） |
| 视角跟随 | 录制中地图中心跟随当前位置移动 |
| 起点标记 | 绿色标记点，标注"起点" |
| 当前位置 | 蓝色圆点实时更新 |

---

### 4.3 历史管理模块

#### 4.3.1 历史列表展示

| 项目 | 规则 |
|------|------|
| 入口 | 首页顶部"历史"按钮 / 底部导航"历史"标签 |
| 列表项内容 | 轨迹名称、日期时间、总距离、总时长、缩略图 |
| 排序 | 按创建时间倒序（最新在前） |
| 操作 | 点击 → 进入轨迹详情/回放；左滑 → 删除 |
| 删除确认 | 弹出确认对话框"确定删除此轨迹？删除后不可恢复" |
| 空状态 | 显示"暂无轨迹记录，去记录第一条轨迹吧"+ 引导按钮 |

#### 4.3.2 单条轨迹地图回放

| 项目 | 规则 |
|------|------|
| 地图展示 | 显示完整轨迹线，自动调整视野包含全部轨迹 |
| 回放控制 | 播放/暂停、进度条、速度切换（1x/2x/4x/8x） |
| 回放动画 | 沿轨迹线移动的标记点，按时间戳顺序播放 |
| 信息面板 | 显示当前回放位置的经纬度、时间、累计距离 |
| 轨迹信息 | 顶部显示：总距离、总时长、平均速度、最高速度 |

#### 4.3.3 本地数据存储规则

| 项目 | 规则 |
|------|------|
| 存储方式 | LocalStorage |
| 存储键名 | `track_records` |
| 存储格式 | JSON 字符串 |
| 单条轨迹上限 | 5000 个点位（超出后自动降频，每 6 秒采集一次） |
| 总存储上限 | LocalStorage 约 5MB，约可存储 50 条平均 100 点的轨迹 |
| 存储时机 | 停止记录时一次性保存；录制中每 30 秒自动保存一次（防丢失） |
| 数据导出 | 支持导出为 GPX 格式文件（可选功能） |

## 5. 非功能性需求

### 5.1 性能

| 指标 | 目标值 |
|------|--------|
| 地图首次加载 | ≤ 3 秒（4G 网络） |
| 定位刷新间隔 | 录制中 3 秒/次 |
| 轨迹绘制延迟 | ≤ 500ms（1000 点以内） |
| 历史列表渲染 | ≤ 1 秒（50 条以内） |
| 回放帧率 | ≥ 30fps |

### 5.2 兼容性

| 平台 | 最低版本 |
|------|---------|
| Android Chrome | 80+ |
| Android WebView | Chrome 80+ 内核 |
| iOS Safari | 14+ |
| Firefox Mobile | 80+ |
| Samsung Internet | 13+ |

### 5.3 存储

| 项目 | 说明 |
|------|------|
| LocalStorage 容量 | 约 5MB（浏览器默认限制） |
| 单条轨迹估算 | 100 点位 ≈ 8KB |

### 5.4 离线能力

## 7. 部署与打包方案

### 7.1 网页部署（免费静态托管）

| 平台 | 说明 |
|------|------|
| GitHub Pages | 免费，支持自定义域名，推荐首选 |
| Vercel | 免费，自动 HTTPS，部署便捷 |
| Netlify | 免费，拖拽部署 |
| Cloudflare Pages | 免费，CDN 加速 |

部署步骤：
1. 将 `index.html`、`manifest.json`、`sw.js`、图标文件上传至托管平台
2. 配置 HTTPS（静态托管平台默认提供）
3. 绑定自定义域名（可选）
4. 在高德开放平台配置 Key 的域名白名单

### 7.2 PWA 添加桌面

1. 用户通过浏览器访问已部署的网页
2. 浏览器触发 `beforeinstallprompt` 事件
3. 用户点击"添加到主屏幕"
4. 桌面生成应用图标，点击后全屏运行

### 7.3 打包 APK 方式

| 方案 | 说明 | 推荐度 |
|------|------|--------|
| Bubblewrap（推荐） | Google 官方 TWA 打包工具，生成原生 APK | ★★★★★ |
| PWABuilder | 微软提供的在线打包服务，一键生成 APK | ★★★★ |
| Apache Cordova | WebView 容器方式，功能完整但体积较大 | ★★★ |
| Capacitor | Ionic 团队维护，类似 Cordova 但更现代 | ★★★ |

**Bubblewrap 打包流程**：
1. 安装 Node.js 和 Bubblewrap CLI：`npm i -g @aspect/aspect-cli`
2. 初始化项目：`bubblewrap init --manifest=https://your-domain.com/manifest.json`
3. 构建 APK：`bubblewrap build`
4. 生成签名 APK，可直接安装到安卓设备

**PWABuilder 打包流程**：
1. 访问 https://www.pwabuilder.com/
2. 输入已部署的 PWA 网址
3. 点击"Build My PWA" → 选择 Android
4. 下载生成的 APK
