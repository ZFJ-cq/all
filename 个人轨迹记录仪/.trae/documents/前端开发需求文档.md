### 2.2 Service Worker 缓存规则（sw.js）

```javascript
const CACHE_NAME = 'track-recorder-v1';

const PRECACHE_URLS = [
  '/',
  '/index.html',
  '/css/style.css',
  '/js/app.js',
  '/js/map.js',
  '/js/tracker.js',
  '/js/history.js',
  '/js/player.js',
  '/js/storage.js',
  '/manifest.json',
  '/icons/icon-192x192.png',
  '/icons/icon-512x512.png'
];

// 安装：预缓存 App Shell
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then((cache) => cache.addAll(PRECACHE_URLS))
      .then(() => self.skipWaiting())
  );
});

// 激活：清理旧版本缓存
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then((names) =>
      Promise.all(
        names.filter((n) => n !== CACHE_NAME).map((n) => caches.delete(n))
      )
    ).then(() => self.clients.claim())
  );
});

// 请求拦截
self.addEventListener('fetch', (event) => {
  const url = new URL(event.request.url);

  // 高德地图 API：网络优先
  if (url.hostname.includes('amap.com')) {
    event.respondWith(
      fetch(event.request).catch(() => caches.match(event.request))
    );
    return;
  }

  // 静态资源：缓存优先 + 后台更新
  event.respondWith(
    caches.match(event.request).then((cached) => {
      const fetchPromise = fetch(event.request).then((response) => {
        caches.open(CACHE_NAME).then((cache) => cache.put(event.request, response));
        return response.clone();
      });
      return cached || fetchPromise;
    })
  );
});
```

**缓存策略**：

| 资源类型 | 策略 | 原因 |
|---------|------|------|
| App Shell（HTML/CSS/JS） | 缓存优先 + 后台更新 | 确保离线可加载 |
| 高德地图 JS API | 网络优先 | API 需在线加载 |
| 地图瓦片 | 不缓存 | 体积过大 |
| 图标资源 | 缓存优先 | 静态不常变 |

**版本更新**：每次发布更新 `CACHE_NAME` 版本号，`activate` 事件自动清理旧缓存。

## 3. 地图初始化参数、定位调用规则、GPS 点位采集逻辑

### 3.1 地图初始化参数

```javascript
const map = new AMap.Map('map-container', {
```

### 3.2 定位调用规则

```javascript
const geolocation = new AMap.Geolocation({
  enableHighAccuracy: true,     // 高精度定位
  timeout: 10000,               // 超时10秒
  maximumAge: 0,                // 不使用缓存位置
  convert: true,                // 自动偏移坐标（GCJ-02）
  showButton: false,
  showMarker: true,
  showCircle: true,
  panToLocation: true,          // 定位后移动到定位点
  zoomToAccuracy: true,
  markerOptions: {
    content: '<div class="location-marker"></div>',
    offset: new AMap.Pixel(-8, -8)
  },
  circleOptions: {
    strokeColor: '#4A90D9',
    strokeOpacity: 0.3,
    strokeWeight: 1,
    fillColor: '#4A90D9',
    fillOpacity: 0.15
  }
});
```

### 3.3 GPS 点位采集逻辑

**核心配置**：

| 参数 | 值 | 说明 |
|------|-----|------|
| collectInterval | 3000ms | 采集间隔 |
| minAccuracy | 100m | 精度阈值，超过丢弃 |
| minDistance | 2m | 最小距离，低于丢弃 |
| autoSaveInterval | 30000ms | 自动保存间隔 |
| backgroundTimeout | 300000ms | 后台超时自动暂停（5分钟） |
| maxPoints | 5000 | 单条轨迹最大点位数 |

**采集流程**：

```
开始记录
  ↓
navigator.geolocation.watchPosition（enableHighAccuracy: true）
  ↓
每次位置更新 → 精度过滤（>100m 丢弃）→ 距离过滤（<2m 丢弃）
  ↓
合格点位 → 加入轨迹数组 → 更新地图轨迹线 → 更新UI信息
  ↓
每30秒自动保存至 LocalStorage（防丢失）
  ↓
页面切至后台 >5分钟 → 自动暂停
  ↓
停止记录 → 计算统计信息 → 保存至历史
```

**距离计算**：使用 Haversine 公式计算两点间球面距离。

## 5. 按钮交互逻辑

### 5.1 状态机

```
IDLE（未录制）──点击"开始记录"──→ TRACKING（录制中）
                                    ↕ 点击"暂停"/"继续"
                                 PAUSED（已暂停）
                                    │
                              点击"停止"→确认
                                    ↓
                              保存轨迹 → IDLE
```

### 5.2 开始记录

- 前置检查：定位权限是否已获取
- 创建新轨迹对象
- 启动 `watchPosition` 监听
- 启动自动保存定时器（30秒）
- 监听 `visibilitychange` 事件
- UI：隐藏"开始记录"按钮，显示"暂停"和"停止"按钮，显示录制信息条

### 5.3 暂停/继续

- 暂停：`clearWatch` 停止监听，按钮变为"继续"
- 继续：重新 `watchPosition`，按钮变为"暂停"

### 5.4 停止记录

- 弹出确认对话框"确定停止记录并保存轨迹？"
- 确认后：停止监听、清除定时器、计算统计信息、保存轨迹、返回首页
- 取消：继续录制

### 5.5 查看历史

- 录制中点击"历史"：提示"录制中，请先停止记录"
- 非录制状态：切换到历史列表视图

## 6. 轨迹绘制、回放绘制逻辑

### 6.1 实时轨迹绘制

| 项目 | 规则 |
|------|------|
| 绘制方式 | `AMap.Polyline` 实时追加坐标点 |
| 线条颜色 | `#4A90D9`（蓝色） |
| 线条宽度 | 6px |
| 线条样式 | 实线，`lineJoin: round`, `lineCap: round` |
| 视角跟随 | 录制中地图中心跟随当前位置 |
| 起点标记 | 绿色标记，标注"起点" |
| 当前位置 | 蓝色圆点实时更新 |

**更新逻辑**：每次新增合格点位 → `polyline.setPath(所有点位坐标)` → 更新当前标记位置 → `map.setCenter(当前坐标)`

### 6.2 历史轨迹展示

- 绘制完整轨迹线（实色）
- 起点标记（绿色"起点"）+ 终点标记（红色"终点"）
- `map.setFitView([polyline])` 自动调整视野包含完整轨迹

### 6.3 轨迹回放逻辑

**回放元素**：
- 半透明底色轨迹线（完整路径，`strokeOpacity: 0.4`）
- 实色已走过轨迹线（动态更新，`strokeOpacity: 1`）
- 移动标记点（沿轨迹移动）

**回放控制**：

| 操作 | 行为 |
|------|------|
| 播放 | 按时间戳顺序逐点移动标记，更新已走过轨迹线 |
| 暂停 | 停止动画，保持当前位置 |
| 继续 | 从当前位置恢复播放 |
| 进度跳转 | `seekPlayback(percent)` 直接跳到指定百分比位置 |
| 速度切换 | 1x/2x/4x/8x，影响每帧前进的点位数和帧间隔 |

**动画帧逻辑**：

```javascript
// 每帧前进点位数 = max(1, floor(speed))
// 帧间隔 = 33ms / speed（约30fps基准）
// 更新：已走过轨迹线 path + 移动标记 position + UI 信息
// 结束：currentIndex >= points.length 时停止
```

### 7.3 index.html PWA 注册代码

```html
<!-- head 中 -->

<meta name="theme-color" content="#1A73E8">

    document.getElementById('install-banner').style.display = 'none';
```

## 8. 打包 APK 技术方案说明

### 8.1 方案一：Bubblewrap（TWA）— 推荐

**原理**：Trusted Web Activity 将 PWA 以全屏 Chrome 渲染方式包装为原生 APK。

**优势**：Google 官方支持、体积小（2-3MB）、性能接近原生、Play Store 可上架。

**步骤**：

```bash
# 1. 安装 JDK 11+ 和 Android SDK
# 2. 安装 Bubblewrap CLI
npm install -g @nicolo-ribaudo/nicolo-nicolo
# 正确包名：@nicolo-ribaudo/nicolo-nicolo
# 实际使用时请查阅 npmjs.com 获取最新包名

# 3. 初始化项目（从已部署的 PWA manifest 拉取配置）
bubblewrap init --manifest=https://your-domain.com/manifest.json

# 4. 构建 APK
bubblewrap build

# 5. 输出：app-release-signed.apk
```

**配置要点**：
- `launchUrl`：指向 `/index.html`
- `themeColor`：`#1A73E8`
- `backgroundColor`：`#FFFFFF`
- `enableNotifications`：false（本项目不需要推送）
- `splashScreenFadeOutDuration`：300

### 8.2 方案二：PWABuilder（在线打包）— 最便捷

**步骤**：
1. 访问 https://www.pwabuilder.com/
2. 输入已部署的 PWA 网址
3. 点击 "Build My PWA" → 选择 Android
4. 下载生成的 APK

**适用场景**：快速测试、无需本地开发环境。

### 8.3 方案三：Capacitor（WebView 容器）

**适用场景**：需要调用原生设备 API（如后台定位、通知等）。

**步骤**：

```bash
# 1. 安装 Capacitor
npm install @capacitor/core @capacitor/cli
npx cap init "轨迹记录" "com.tracker.app"

# 2. 添加安卓平台
npm install @capacitor/android
npx cap add android

# 3. 复制 Web 资源
npx cap copy

# 4. 用 Android Studio 打开构建
npx cap open android
```

### 8.4 方案对比

| 方案 | 体积 | 性能 | 原生能力 | 上架 Play Store | 复杂度 |
|------|------|------|---------|----------------|--------|
| Bubblewrap (TWA) | 2-3MB | 高 | 有限 | 可 | 中 |
| PWABuilder | 2-3MB | 高 | 有限 | 可 | 低 |
| Capacitor | 10-15MB | 中 | 完整 | 可 | 高 |

**推荐**：个人自用首选 Bubblewrap 或 PWABuilder，简单快速。
